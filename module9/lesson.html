Module 9: Embedding Media

Introduction to HTML — Rebuilding the Space Jam Website

Goal: Build two media pages from the Space Jam site: the Planet B-Ball QTVR viewer page, which uses the <embed> tag to play an inline QuickTime VR file, and a Looney Tunes voice clip download page, which links to audio files for the user to save.

The embed Tag

The <embed> tag inserts a media file directly into the page, played by whatever browser plugin supports that format. In 1996, this was primarily used for QuickTime movies, QuickTime VR panoramas, and Macromedia Director/Shockwave content.

<embed src="assets/centercourt.qt" width=320 height=200 autoplay=true>

Key attributes:

  • src — path to the media file
  • width and height — dimensions of the embedded player in pixels
  • autoplay — set to true to start playing immediately on load
QuickTime VR files require the QuickTime browser plugin, which no modern browser supports. The embed tag will render an empty placeholder in current browsers. This is expected — the markup is correct, the technology is obsolete.

Linking to Downloadable Files

Linking to a media file works exactly like linking to an HTML page — use the <a href> tag with the file's path as the destination. When the browser receives a file type it cannot display inline, it prompts the user to download it.

<a href="assets/growl.wav">growl.wav (12k)</a>

In 1996, it was common practice to offer the same audio clip in multiple formats for different operating systems: AIF for Mac users, WAV for Windows users. The same link pattern applied to both — just different file paths.

<a href="assets/growl.aif">AIF</a> | <a href="assets/growl.wav">WAV</a>

Lists for Download Pages

Download pages in the Space Jam site use an unordered list where each item is a clip name followed by inline links for each available format. The <li> tag marks each entry, and the format links sit inline after the title.

The hr Tag

The <hr> tag draws a horizontal rule — a dividing line across the page. It takes no closing tag. Attributes include width (as pixels or percentage) and noshade (renders as a flat line instead of an embossed one).

<hr width="50%" noshade>

Steps: Part A — The QTVR Viewer

Step 1 — Create centercourt.html

Create a new file. Title it "Planet B-Ball". Set the body color scheme: bgcolor="#18ad11", background="assets/bg-bball.gif", text="#030762", link="#050cb2", vlink="#050cb2", alink="#050cb2". Notice this section uses a completely different palette from the rest of the site.

Step 2 — Add the JordanDome header and embed

Inside the body, add a centered b-jordandome.gif image. Below it, in a blockquote, add a brief intro sentence in <font size="+1">, then a centered <embed> tag pointing to the QuickTime VR file: src="assets/centercourt.qt" (note: this file is not in assets — it is large), width=320, height=200, autoplay=true.

The .qt file is not included in the assets folder due to its size. The embed tag will show a placeholder. That is correct.

Step 3 — Add usage instructions and download links

Below the embed, add instructions for how to use the QTVR (mouse to look around, control to zoom out, shift/option to zoom in). Then add a sentence explaining that users without the plugin can download the file, with links to the .sea.hqx (Mac) and .zip (Windows) versions in the fullsite's dld/qtvr/ folder.

Step 4 — Add the navigation footer

Close the blockquote and add a centered footer using the same pattern as Module 4: qtvr.gif linking back to qtvr.html, the copyright notice, and next.gif linking to gym.html.

Steps: Part B — The Audio Download Page

Step 5 — Create sounds.html

Create a new file. Title it "Space Jam - Sounds". Set the body: bgcolor="#000000", text="#ff0000", link="#ff4c4c", vlink="#ff4c4c".

Step 6 — Add the heading and hr

Add a centered heading "Looney Tunes Voice Clips" followed by an <hr> with width="75%" and noshade.

Step 7 — Add the download list

Create an unordered list. Each <li> should contain the character name, a description of the clip in quotes, and two inline links: one for AIF and one for WAV. Use the six clips in the assets folder. For example:

Bugs Bunny — "Yeah!" — [AIF] [WAV]

Where [AIF] and [WAV] are anchor tags pointing to the respective files in assets/.

Step 8 — Add a closing hr and footer

Add a second <hr width="50%" noshade> and a copyright footer.

What You Built

Your finished files should match completed/centercourt.html and completed/sounds.html. These two pages demonstrate the two ways the Space Jam site delivers media: embedded inline (QTVR) and as downloadable files (audio clips).
