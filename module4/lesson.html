Module 4: Links

Introduction to HTML — Rebuilding the Space Jam Website

Goal: Add working navigation links to the Bugs Bunny bio page, then create a second bio page for Daffy Duck and a simple Looney Tunes bios index. The three pages will link to each other, forming a mini navigable section.

The a Tag

Links are created with the <a> tag. The href attribute (short for hypertext reference) specifies where the link goes. Whatever sits between the opening and closing tag becomes the clickable content.

<a href="daffy.html">Go to Daffy Duck</a>

Links do not have to wrap text. They can wrap images too — clicking the image navigates to the URL.

<a href="daffy.html"><img src="assets/next.gif" border=0></a>
Always set border=0 on images inside links. Without it, browsers draw a colored rectangle border around the image to indicate it is clickable, which almost always looks wrong.

Relative vs. Absolute Paths

A relative path is relative to the current file's location. If bugs.html and daffy.html are in the same folder, the link is simply:

<a href="daffy.html">

An absolute path includes the full URL, starting with https://. Use this for linking to other websites:

<a href="https://www.warnerbros.com/">Warner Bros.</a>

To navigate up a folder level, use ../. To navigate into a subfolder, include the folder name: subfolder/page.html.

The target Attribute

The target attribute controls where the linked page opens.

  • target="_blank" — opens in a new browser tab or window
  • target="_top" — breaks out of any frames and loads the page in the full browser window (used throughout the Space Jam site)
<a href="https://www.warnerbros.com/" target="_blank">Warner Bros.</a>

Lists

The bios index page will use an unordered list to display character names. An unordered list is created with <ul>, and each item in it uses <li>.

<ul>
  <li><a href="bugs.html">Bugs Bunny</a></li>
  <li><a href="daffy.html">Daffy Duck</a></li>
</ul>

Steps

Step 1 — Add links to bugs.html

Start from your completed bugs.html from Module 2. Find the navigation footer at the bottom. The two image buttons currently have no link wrapper. Wrap each one in an <a> tag:

  • The bios.gif image should link to looneybios.html
  • The next.gif image should link to daffy.html

Step 2 — Create daffy.html

Create a new file called daffy.html in the same folder as bugs.html. Use the same skeleton and structure from Module 2 but with Daffy's content. The title, images, bio text, and footer all follow the same pattern — just with Daffy's assets and text.

In Daffy's footer, the bios.gif links back to looneybios.html, and the next.gif links forward to porky.html (which does not need to exist yet).

Step 3 — Create looneybios.html

Create a simple index page called looneybios.html. It should have the title "The Lineup", a heading "Looney Tunes Bios", and an unordered list linking to each bio page. For now just link to bugs.html and daffy.html.

Step 4 — Test the links

Open looneybios.html in your browser. Click Bugs Bunny — you should land on the bio. Click the Next button — you should arrive at Daffy's page. Click Bios — you should return to the index. All three pages are now linked together.

What You Built

Compare your pages to the completed/ folder. You now have a working mini-site: three pages that navigate between each other. This is the same link structure the entire Space Jam Lineup section is built on.
