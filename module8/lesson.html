Module 8: Frames

Introduction to HTML — Rebuilding the Space Jam Website

Goal: Reconstruct the two-frame layout used by every section of the Space Jam site: a narrow persistent navigation column on the left, and a main content area on the right that changes as the user navigates.

What are Frames?

Frames divide the browser window into independent panes, each displaying a separate HTML file. This was a popular pattern in 1996 for keeping navigation persistent while the content area changed. Every section of the Space Jam site uses a two-column frameset: a 100-pixel-wide nav column on the left and the content to the right.

Frames are not used in modern web development — they have been removed from the HTML standard. We are implementing them here because the Space Jam site is built on them and they are part of what makes it historically interesting.

frameset and frame

A frames page replaces <body> with <frameset>. The frameset defines how the window is divided, and each <frame> inside it points to an HTML file to display.

The cols attribute splits the window into columns. The rows attribute splits it into rows. Values can be pixel widths, percentages, or * to mean "the rest of the space."

<frameset cols="100,*">
  <frame name="nav" src="nav.html">
  <frame name="main" src="content.html">
</frameset>

frame Attributes

  • name — gives the frame a name so links can target it
  • src — the HTML file to load into this frame
  • scrolling — "yes", "no", or "auto"
  • noresize — prevents the user from dragging the frame border to resize
  • marginwidth and marginheight — inner margin of the frame in pixels

Targeting Frames with Links

A link in the nav frame can load a page into the main content frame by using the content frame's name as the link's target:

<a href="jamcentral.html" target="main">Jam Central</a>

Two special target values are also important:

  • target="_top" — breaks out of all frames and loads the page in the full window
  • target="_blank" — opens in a new browser tab

noframes

The <noframes> tag provides fallback content for browsers that do not support frames. It goes inside the <frameset> and contains a <body> with a message and a link to the non-frames version of the page.

<noframes>
  <body>
    Your browser does not support frames.
    <a href="jamcentral.html">Click here.</a>
  </body>
</noframes>

Steps

Step 1 — Create the frameset file

Create jamcentralframes.html. This file has a <head> with the title "Jam Central", but instead of <body> it uses <frameset cols="100,*">.

Inside the frameset, add a <noframes> block with a message and a link to jamcentral.html. Then add two <frame> tags:

  • First frame: name="nav", src="jamcentralnav.html", scrolling="auto", noresize, marginwidth=0, marginheight=0
  • Second frame: name="main", src="jamcentral.html", scrolling="auto", noresize, marginwidth=0, marginheight=0

Step 2 — Create the nav file

Create jamcentralnav.html. This is a regular HTML page with a black starfield body and white text. It contains a centered vertical stack of linked images — one per section — separated by line breaks. Every link uses target="main" so the page loads in the content frame, not inside the nav frame.

Use the assets listed above. The nav images and their dimensions:

  • pn-jamcentralanim.gif — 70×49, links to jamcentral.html
  • pn-behind.gif — 70×49, links to behind.html
  • pn-junior.gif — 70×44, links to junior.html
  • pn-lineup.gif — 70×41, links to lineup.html
  • pn-lunar.gif — 70×54, links to tunes.html
  • pn-bball.gif — 70×47, links to bball.html
  • pn-souvenirs.gif — 70×56, links to souvenirs.html
  • pn-jump.gif — 70×40, links to jump.html
  • pn-pressbox.gif — 70×44, links to pressbox.html
  • credits.gif — 35×35, links to credits.html
  • malogosm.gif — 70×50, links to http://www.marsattacks.com/ with target="_blank"

Step 3 — Reuse the jamcentral.html content file

Copy the jamcentral.html you built in Module 7 into the same folder. No changes needed — it will load automatically as the initial content of the main frame.

Step 4 — Open and test

Open jamcentralframes.html in your browser. You should see the narrow nav column on the left and the Jam Central hub on the right. Click a nav button — the URL it links to will attempt to load in the right frame. (Most target pages don't exist yet, which is expected.)
